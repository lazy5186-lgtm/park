========================================
  N_blog_auto - 빌드 & 배포 명령어 정리
========================================

[사전 준비]
- Node.js 설치 필요
- 프로젝트 폴더에서 터미널(cmd/PowerShell) 실행


----------------------------------------
1. 의존성 설치
----------------------------------------
npm install


----------------------------------------
2. 로컬 테스트 실행
----------------------------------------
npm run start


----------------------------------------
3. EXE 빌드 (Windows 설치파일 생성)
----------------------------------------
npm run build

- 결과물: dist/ 폴더에 생성됨
  - N_blog_auto Setup x.x.x.exe  (설치파일)


----------------------------------------
4. Git 기본 명령어
----------------------------------------

# 현재 상태 확인
git status

# 모든 변경사항 스테이징
git add .

# 커밋
git commit -m "커밋 메시지"

# 원격 저장소에 푸시
git push origin main


----------------------------------------
5. Git 버전 태그 + 자동 업데이트 배포
----------------------------------------
(electron-updater가 GitHub Releases를 확인하므로
 새 버전 배포 시 태그를 올려야 자동 업데이트 작동)

# 1) package.json의 version 수정 (예: 1.0.0 -> 1.1.0)

# 2) 커밋 + 푸시
git add .
git commit -m "v1.1.0 업데이트 내용"
git push origin main

# 3) 태그 생성 + 푸시
git tag v1.1.0
git push origin v1.1.0

# 4) 빌드
npm run build

# 5) GitHub Releases에 빌드 결과물 업로드
#    dist/ 폴더의 아래 파일들을 GitHub Release에 첨부:
#    - N_blog_auto Setup x.x.x.exe
#    - latest.yml


----------------------------------------
6. 한줄 요약 (빠른 배포)
----------------------------------------

# 버전 올리고 빌드 + 푸시 + 태그 한번에
git add . && git commit -m "v1.1.0 업데이트" && git push origin main && git tag v1.1.0 && git push origin v1.1.0 && npm run build


----------------------------------------
7. 기타 유용한 명령어
----------------------------------------

# 빌드 결과물 삭제
rm -rf dist/

# Git 로그 확인
git log --oneline -10

# 태그 목록 확인
git tag -l

# 태그 삭제 (잘못 만든 경우)
git tag -d v1.1.0
git push origin --delete v1.1.0
